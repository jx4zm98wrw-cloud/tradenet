<DocsH2 num="01" id="n-overview">The 45-class split</DocsH2>

Classes 1–34 cover <strong>goods</strong> (tangible products). Classes 35–45 cover <strong>services</strong>. The split matters because conflict analysis usually starts with: are these marks in the same or related classes? A pharmaceutical (Class 5) and a restaurant (Class 43) sharing the same name rarely conflict; two food brands in Class 30 almost always do.

<DocsH2 num="02" id="n-common">Commonly confused classes</DocsH2>

Three pairs that trip up filers and create avoidable opposition windows:

<ul className="docs-list">
  <li><strong>Class 35 (Retail services) vs. the class of the goods sold.</strong> A clothing brand selling its own line files in Class 25 (the clothes). A department store selling many brands' clothes files in Class 35.</li>
  <li><strong>Class 9 (Software) vs. Class 42 (SaaS / cloud).</strong> Downloaded apps go in Class 9. Hosted, browser-delivered services go in Class 42. Many filings need both.</li>
  <li><strong>Class 5 (Pharmaceuticals) vs. Class 44 (Medical services).</strong> A drug is a Class 5 good. A clinic that prescribes it is a Class 44 service. Often filed together for healthcare brands.</li>
</ul>

<DocsCallout heading="Cross-class conflict">
  Vietnamese examiners sometimes find conflict across “related classes” even without direct overlap — e.g. Class 32 (beverages) and Class 33 (alcoholic beverages). Tradenet surfaces these via the semantic-similarity channel in the conflict scorecard.
</DocsCallout>

<DocsH2 num="03" id="n-search">Searching across classes</DocsH2>

In the search rail, the Nice classes group has an <strong>ANY / ALL</strong> toggle. Default is ANY — match marks covering any selected class. Switch to ALL when you need marks that cover the full set (e.g. food brands that cover Classes 29, 30, <em>and</em> 32 together).
