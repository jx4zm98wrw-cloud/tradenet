<DocsH2 num="02" id="None">Search</DocsH2>

<span className="mono"><strong>POST</strong> /v1/search</span> — run a one-off search. Returns up to 200 results per call; paginate via the <span className="mono">cursor</span> field.

<DocsCode lang="JSON">{`POST /v1/search
{
  "mode": "phonetic",
  "q": "neur*",
  "filters": {
    "classes": [5, 10],
    "countries": ["VN", "SG"],
    "period": { "from": "2026-01-01" }
  },
  "threshold": 0.75,
  "sort": "similarity",
  "limit": 50
}`}</DocsCode>

<DocsH2 num="03" id="None">Watchlists</DocsH2>

<span className="mono"><strong>POST</strong> /v1/watchlists</span> creates a saved query that re-runs each gazette issue. Findings are delivered via webhook, email, or by polling the watchlist endpoint.

<DocsH2 num="04" id="None">Webhooks</DocsH2>

Subscribe to events at <span className="mono">POST /v1/webhooks</span>. All webhook payloads are signed with HMAC-SHA256 — verify the <span className="mono">X-TM-Signature</span> header.

  headers={["Event", "Triggered when"]}

  ]}

<DocsH2 num="05" id="None">Rate limits</DocsH2>

<ul className="docs-list">
  <li><strong>Search:</strong> 100 requests/min · 10,000/day</li>
  <li><strong>Watchlists:</strong> no rate limit on writes; reads paginated</li>
  <li><strong>Webhook delivery:</strong> at-least-once, exponential backoff up to 24h</li>
</ul>
