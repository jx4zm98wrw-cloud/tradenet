<h2 className="docs-h2" id="v-what">
  <span className="docs-h2-num">01</span> What Vienna codes are
</h2>

<p className="docs-p">
  Vienna codes classify the <strong>figurative content</strong> of a mark — what you see in it, not what it spells. A logo containing a stylized “V” and a circle gets coded against two categories simultaneously. The same mark might carry 5–15 codes in a thorough classification.
</p>

<p className="docs-p">
  The classification is maintained by WIPO and revised every few years. Tradenet uses <strong>Vienna 5.1</strong> (2023 revision). Older marks classified under 4.x or 3.x are auto-mapped where the categories are equivalent; flagged for review where they aren't.
</p>

<h2 className="docs-h2" id="v-format">
  <span className="docs-h2-num">02</span> Reading the code format
</h2>

<p className="docs-p">
  Codes are three-level and dotted: <span className="mono">CC.DD.SS</span> — Category, Division, Section. For example, <span className="mono">26.04.04</span> means category 26 (geometric figures), division 04 (quadrilaterals), section 04 (rectangles).
</p>

<DocsCallout heading="Auxiliary codes">
  A leading <span className="mono">A</span> on the last digit (e.g. <span className="mono">26.04.04 A.1</span>) is an auxiliary code — finer-grained sub-classification that's optional but improves recall in image search.
</DocsCallout>

<h2 className="docs-h2" id="v-search">
  <span className="docs-h2-num">03</span> Searching by Vienna code
</h2>

<p className="docs-p">
  On the Search page, switch to the <strong>Vienna code</strong> mode tab. You can search by a single full code, a partial code (e.g. <span className="mono">26.04</span> matches everything under category 26 / division 04), or a combination with boolean operators.
</p>

<DocsCode lang="DSL">{`# Find all marks combining a stylized V with a circle
